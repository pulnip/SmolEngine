## 파일 구조 관례



## 전역 코드 관례

따로 명시하지 않는 이상, **Engine/**과 **Source/**의 코드들은 이 관례를 상속받습니다.


### 기본 설정

- **들여쓰기**: 공백(Space) 4칸. (탭(Tab) 문자 금지.)
- **UTF-8**: UTF-8 이외의 문자열로 변경 후 저장 금지.


### 헤더 규칙


### 명명 규칙

- **헤더 파일 이름**: C와의 호환성이 있는 헤더는 *.h*, C++ 전용 헤더는 *.hpp*
- **클래스 이름**: `PascalCase`
- (권장) **`bool` 타입**: `is-`혹은 `has-` 접두사를 가질 것.


### 변수 규칙

- `BOOL`, `NULL` 등 대체제가 있는 c-style 금지.
- `std::wstring`, `wchar_t` 금지. (가능하면 `std::string`, `char` 사용할 것.)
    - (예외) `std::filesystem::path`에서, `L"..."` literal 허용.


### 함수 규칙

- **Early Return** 적극 권장.


### 클래스 규칙

- `struct`에 `virtual` 키워드나 상속 금지, **데이터 모음** 용도로 사용할 것.
- `class`는 RAII를 만족할 것. (*보이스카웃 룰*)


---

## **Engine/**내에서의 관례

- **Source/**에 의존하지 않도록 만들 것.


### 변수 규칙

- `CHAR`, `WCHAR`, `TCHAR` 등 OS 전용 타입 금지.
- (권장) 크기가 타입에 명시된 타입 사용.


### 클래스 규칙

- `class`는 **The Rule of Five**를 만족할 것. (기본 생성자들에 대해서 `Semantics.hpp` 사용 권장.)
- Public 헤더에 있는 클래스는 **Pimpl 관례**를 사용할 것.


### 리소스 관리 규칙

- 특별한 이유를 제외하고 `new` 금지.
- `std::unique_ptr`에 명시적 포인터 전달 금지. (`std::make_unique`를 선호할 것.)
- COM 객체는 반드시 `ComPtr`로 관리.
- 리소스 핸들을 유저 영역에 반환 금지.


---

## **Source/**내에서의 관례

### 헤더 파일 규칙

- 표준 라이브러리를 제외하고, **Enigine/**을 통하지 않은 외부 라이브러리 직접 include 금지.


### 리소스 관리 규칙

- 어떠한 경우에도 `new` 금지, 엔진에서 제공한 타입에 대한 임의 `delete` 금지.
